Shortest Remaining Time First (SRTF) Algorithm

Description

The SRTF (Shortest Remaining Time First) algorithm selects the process with the smallest remaining burst time from the ready queue for execution. It is a preemptive scheduling algorithm.

Processes are scheduled based on their remaining burst times, where the process with the smallest remaining time is given priority. If a new process arrives with a shorter burst time than the currently running process, the currently running process is preempted and the new process is executed.

Problem

Process Arrival Time Burst Time
P1 0 9
P2 1 5
P3 2 4

Solution

Step-by-Step Execution

  1. Process Arrival:
    • At time 0, only Process P1 is available in the ready queue.
    • At time 1, Process P2 arrives.
    • At time 6, Process P3 arrives.
  2. Process Execution:
    • Process P1 starts execution at time 0 and runs until time 1.
    • At time 1, Process P2 arrives and runs until 2, now P2 and P3 have same burst time. As P2 is already executing, it will continue its execution till 6 and completes its task.
    • At time 6, Process P3 arrives with a shortest burst time and runs till 10 and completed its job.
    • Now the left over process if P1, it will execute for its remaining time and run till 18 and completes its job.
  3. Waiting Time:
    • Waiting Time of P1 = (0 - 0) + (10 - 1) = 9
    • Waiting Time of P2 = 1 - 1 = 0
    • Waiting Time of P2 = 6 - 2 = 4
  4. Turnaround Time: (Waiting Time + Burst Time)
    • Turnaround Time of P1 = 9 + 9 = 18
    • Turnaround Time of P2 = 0 + 5 = 5
    • Turnaround Time of P3 = 4 + 4 = 8
  5. Average Waiting Time: (9 + 0 + 4) / 3 = 4.33
